#############################################
Transect and discharge
#############################################

This screen allows the user to chose where to put the vector on which the river discharge will be processed.
As you can see, the widget used to draw the transect is the same as the one used to configure the beacons positions (ShapeOnImage).

When the user has placed the vector as wanted, he can click on the "Display transects" button to start the post processing step.
When the post processing is done, the current widgets are replaced with the processed image showing the discharge vectors, as well as the calculated river discharge.

As other screens, when entering the screen, the application checks if the step is already done. If so, it only displays the processed image and the know river discharge value.
If not, it calls the ``load_transect_selection`` method to load the image and the selection widget.

When the user presses the button to launch the post processing, the application calls the ``process_and_plot_transects`` function that will have the same behavior as the ``generate_piv`` function.
Indeed, it will check if the application has connexion to the internet, and if so it will call the ``/process-transects`` route on the API.
This route will process the transects and return the processed image and the river discharge value.

If the user is not connected on the internet, or you are not using the api version, the same process will be done locally.

In both ways, the application will use the "piv.nc" file generated by the previous screen to process the velocity vectors field and calculate the river discharge.

To sum up the functions used in this screen:

- ``load_transect_selection``: Load the image and the selection widget.
- ``process_and_plot_transects``: Process the transects and plot the river discharge.

This screen calls back function from src/back/transect.py, which will step-by-step process the transects and calculate the river discharge.
An very important point is the local_points parameter in the ``delimiter_points_bathy`` function, these points will define the vector on which the river discharge will be calculated. Before this year it was hardcoded, but in no way it can be done like that.
